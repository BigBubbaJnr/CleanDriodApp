My answer to Replit's question first

Should we implement the background task before Play Store submission, or soft-launch without it?

My recommendation is:

Soft-launch without it.

Here's why.

A scheduling feature is not core to what makes CleanDroid valuable.

The core value is:

Storage Intelligence
Junk Cleaner
Duplicate Finder
Large File Scanner
Trustworthy recommendations

Those are the reasons someone downloads the app.

Background scheduling is a convenience feature.

Android's background execution rules have become increasingly strict, and getting background work wrong can create battery issues, inconsistent behaviour, or Play Store review problems. Rather than delaying launch or risking a rejection over a non-essential feature, I'd either:

disable the scheduling UI temporarily, or
clearly label it as "Coming Soon" if you want users to know it's planned.

Then implement it properly for a v1.1 update after you have real users and more testing.
